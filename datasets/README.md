# datasets/ — tracked per-dataset state (the model)

Session data dirs are gitignored (raw frames; several are third-party sets
that must never be committed), so everything the repo must VERSION about a
dataset lives here, keyed `datasets/<session>/<set>/`. **Blank slate after
the x86 reset** — the old dataset instances were cleared (their image data
is backed up off-repo; the process facts are in git history at the
`checkpoint` commit). New datasets are added here as they are processed on
x86.

The per-dataset state MODEL (durable — a new dataset gets these files):

| file | role | status |
|---|---|---|
| `acquisition.json` | how the set was SHOT: `exif` (camera/lens/focal/exposure/ISO/FOV+pixel-scale/cadence — auto-derived by `scripts/lib/acquisition.py`, do not hand-edit) + `mount` (`fixed` \| `tracked` — the one fact EXIF can't record, human-declared). A consumer needing the camera model (e.g. the anomaly audit's cross-frame linking) resolves this FIRST and STOPS if `mount` is undeclared — no silent default | DURABLE — `acquisition.resolve()` seeds it + gates on it |
| `audit_work/anomaly_audit.json` | the obstruction-audit record (per-frame classifications, capture runs, unique linked objects) written by `scripts/qa/anomaly_audit.py` — kept here so the raw image dir stays clean. Its `audit_work/` sibling scratch (Siril residuals / star-lists / `.ssf`) is gitignored | DURABLE — the versioned audit result; regenerated by a re-run |
| `qa_work/frame_metrics.json` | per-frame **frame-quality measurement** — Siril `register` regdata (FWHM / roundness / background / #stars) + robust-z cull candidates, written by `scripts/qa/inspect_stage.py` + `cull_report.py`; kept here so the raw image dir stays clean, its `qa_work/` scratch (CFA FITS / `.seq` / `.ssf`) gitignored. This is the MEASUREMENT; the cull DECISION is a `recipe.json` `stack` policy block adopted on a with/without ladder | DURABLE — the versioned QA result; regenerated by a re-run |
| `geometry.json` | per-set composition facts: terrestrial `foreground` (rect \| npz `mask` path, session-relative), `judgment_crops`, optional `starsep` overrides | DURABLE — resolved by `astrometrics.configure()` in every product entry point |
| `composition.json` | how a multi-line/multi-filter target's composed linear stack is BUILT: `kind: dualband-osc` (CFA line extraction) or `mono-filters` (`members` = channel → sibling per-filter SET, `reference` = the un-interpolated member), plus the `channels` R/G/B palette mapping. Build-side facts only. Absent = ordinary single stack (degrade-loudly) | DURABLE — `run_pipeline.sh` + `compose.py` |
| `recipe.json` | the per-dataset processing knobs + `spcc` spec (broadband sensor/filter names, or narrowband `rwl`/`gwl`/`bwl` wavelengths) + optional `frame_qa` / `stack` policy blocks | MIXED — the `spcc`/`frame_qa`/`stack` blocks are durable; the **`render` knob block is PENDING the x86 chain's schema** (the old schema died with the wiped chain) |
| `experiments.jsonl` | the tuning-experiment ledger (append-only: param, values, control, hypothesis, pinned stack, verdict). A killed hypothesis is ALSO written to the dead-end registry (`docs/dead-ends.md`) with its mechanism | DURABLE model — re-wired to the x86 chain's ladder |
| `baseline.json` | the no-regression record (pinned stack sha, expected gate/shell metrics, artifact hashes) | PENDING — written by the re-ported no-regression sweep on x86; chain-coupled, so none exist yet |
| `../GENERIC.json` | the repo-wide base layer of generic render knobs | STUB — pending the x86 chain re-seeding its knob set |

Rules (the same contract as README "How a change is accepted"):

- `acquisition.json`'s `mount` has **no safe default** (unlike a recipe, which
  falls back to GENERIC): a consumer that needs fixed-vs-tracked STOPS and asks
  rather than assume it. `exif` is tool-written and refreshed; `mount` is the
  human field.
- A dataset with **no recipe** processes with the GENERIC defaults and says
  so loudly — generic is honest, not approved. It never inherits another
  dataset's look.
- An **approved** recipe pins every knob it means to pin (a later generic
  change must not silently restyle an approved render). A provisional recipe
  pins only what it means to.
- `baseline.json` is measurement output, never hand-written; it is recorded
  by the no-regression sweep after an approved change, then git-tagged.
- Geometry mask files (`*.npz`) are DERIVED data — they live in the session
  dir (`work/`), regenerated by `scripts/geometry/suggest_foreground.py`;
  `geometry.json` only points at them.

The render-knob schema, the resolution tooling (`resolve_recipe`), and the
sweep re-establish with the x86 render chain — see `docs/x86-empirical-test-plan.md`.
